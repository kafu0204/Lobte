1.1 Java简介

  • 1.1.1 Java的发展简史、三大版本和优势
    • Java之父：James Gosling(詹姆斯·高斯林)
    • 历史简介：1990年Sun公司成立了由James Gosling领导的项目组，他们在C++基础上开发了一种新的编程语言：Java。后来Java在互联网领域大获成功，而James Gosling后来就被称为Java之父。
    • Java名称的由来：是Java创始人员团队中其中一名成员由于灵感想到的，想起自己在Java岛（爪哇岛）上曾喝过一种美味的咖啡，于是这种计算机编程语言就命名为Java。
    • Java经历的版本：
      • 1995年Java诞生
      • 1996年JDK1.0发布
      • 1998年JDK1.2发布(称为Java2)，随后Java被分为J2SE,J2EE,J2ME
      • 2000年JDK1.3、1.4发布
      • 2004年JDK1.5发布，Sun公司将其更名为JDK5.0
      • 2006年JDK6正式发布(Sun公司发行的最后一个版本)
      • 2010年被美国Oracle公司收购，并且由于Oracle公司并不重用詹姆斯，所以之后詹姆斯离开了Oracle公司
      • ….
    • Java三大版本:
      • Java EE(Java Platform，Enterprise Edition，Java平台，企业版，也称J2EE，为什么叫J2EE：因为是1998年发布JDK1.2的时候分为了三个版本，所以2的意思代表版本1.2的2)作用：帮助开发和部署可移植、健壮、可伸缩且安全的服务器端 Java 应用程序，是用来做网站的。
      • Java SE(Java Platform，Standard Edition，Java平台，标准版，也称J2SE)作用：用于开发和部署桌面、服务器以及嵌入设备和实时环境中的Java运用程序，是做电脑上运行的软件。
      • Java ME：(Java Platform，Micro Edition，Java微版，也称J2ME)作用：是一个技术和规范的集合，它为移动设备提供了Java环境的开发与应用平台。是用来做手机软件的。
    • Java的优势
      • 跨平台的
      • 安全的
      • 完全面向对象的
      • 简单的
      • 健壮的
  • 1.1.2 Java的跨平台原理与运行过程
    • 跨平台原理：
      • Java源代码(编写后的Java代码)
      • 编译器(源代码被编译器编译成字节码文件)
      • 然后由各大平台的虚拟机JVM(Java Virtual Machine)运行这些编译后的字节码文件(Windows、Unix、Linux、MacOSX)
      • 这就做到了一次编写，多处运行。
    • 运行过程：
      • AAA.java
      • 编译器
      • AAA.class
      • JVM(01010110…..)
      • 由计算机运行
    • Java是不是可以在所有平台上运行呢：并不是。因为Java只能在已经提供并安装了Java虚拟机的平台上运行，没有安装虚拟机的的平台是不能运行的。

1.2 Java开发环境

  • 1.2.1 Java开发工具：记事本、IDE(Integrated Development Environment，集成开发环境，集成了代码编写功能、分析功能、编译功能、调试功能等一体化的开发软件服务套。我们学习期间：使用MyEclipse与Eclipse两种集成开发软件。)、反编译工具。
  • 1.2.2 Java程序开发环境JDK：JDK(Java Development Kit，Java 语言的软件开发工具包，JDK是整个Java开发的核心，它包含了Java的运行环境（JVM+Java系统类库）和Java工具，没有JDK的话，无法编译Java程序)，获取JDK：Oracle官方网站免费(http://www.oracle.com)下载。
  • 1.2.3 Java开发环境变量配置(也称配置JDK)（具体过程见文档：下载、安装与配置JDK.docx）：
    • 首先获取JDK之后，安装JDK。直接点击下一步默认安装路径(可以自己修改，但是建议默认，不修改)。会安装两次，一个是JDK，另一个是JRE。
    • 配置JDK(目的是让系统认识)：
      • 右键我的电脑（此电脑/计算机）
      • 点击属性
      • 点击高级系统设置
      • 点击环境变量
      • 用户变量只对当前登录用户有效，系统变量对计算机有效(一般情况下改变系统变量)
      • 新建
      • 配置：
        • 变量名：JAVA_HOME
        • 变量值：JDK版本的安装的根目录，比如我的安装根目录是C:\Program Files\Java\jdk1.8.0_161
      • 在系统变量里面找到Path，双击或者点击编辑
      • 将光标（鼠标）移动至变量值的最前面 输入：%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin; 一定记住：后面要加上分号(win10可以直接新建一栏直接写上路径，不需要封号)
    • 配置CLASSPATH：
      • 继续上述步骤
      • 新建
      • 配置：
        • 变量名:CLASSPATH
        • 变量值：.;%JAVA_HOME%\lib;%JAVA_HOME%\lib\tools.jar;
    • 测试是否配置成功：
      • 搜索cmd，打开
      • 输入java或者javac，然后回车，如果出现一大串英文则代表配置成功。（如果出现’java’或者’javac’ 不是内部或外部命令，也不是可运行的程序或批处理文件。则配置失败。）

1.3 第一个Java程序hello World

  • 1.3.1 程序开发步骤：

    • 源文件XXX.java
    • javac XXX.java
    • 编译后的文件XXX.class
    • 运行java XXX

  • 1.3.2 编写代码内容：编写java源文件，先建立一个名为：Test.java的文件，然后右键编辑，写入如下代码：写代码之前注意以下几点：

    • Java代码是严格区分大小写的。创建Java文件，文件名首字母必须大写，不能用中文。写中文编译会通过，但是极容易出现乱码问题，所以不要用中文命名。

    • Java文件中的代码全部写在一个类中，类名必须跟Java文件名一模一样。

      public class Test {
    public static void main(String[] args) {
        System.out.println("Hello World!"+"\t"+"Hello World");
        System.out.print("Hello World!"+"\n"+"Hello World");
        System.err.println("error!");
    }
}

    • 其中：

      • public：访问权限
      • class：关键字，代表这是一个类
      • Test：类名，要与文件名一致，且首字母一定要大写
      • …main….：程序入口
      • System……：程序代码，打印语句
      • System.out.println()：输出后换行
      • System.out.print()：输出后不换行
      • System.err.println()：输出显示为红色
      • \t：制表符，相当于按一次Tab键
      • \n：换行符

  • 1.3.3 cmd命令窗口执行Java文件步骤（具体图文见文档：编译与执行Java文件步骤.docx）：

    • 点击开始(wn10系统点击左下角右键)，或者Win+R
    • 点击运行
    • 输入cmd，回车
    • 找到文件路径，比如这里是e: 回车，这里的e不区分大小写
    • 如果文件是属于某个文件夹下，则继续输入:cd 文件的路径 如果还有内层文件夹，则继续这一步操作，也可以直接输入cd 文件全路径 直接在D盘或者某盘路径下则跳过此步。或者直接cd Java文件的全路径。
    • 编译java文件，输入：javac Test.java 生成一个Test.class文件
    • 输入：java Test 输出结果.
    • cmd中文件夹之间的操作
      • 要退出当前文件夹，可以使用“cd ..”命令。
      • 如果要直接返回到E盘的根目录，可以输入“cd\”，然后回车
      • 如果要直接转到其他分区，比如F，可以直接输入要转入的盘符，然后回车即可。
